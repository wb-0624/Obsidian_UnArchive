# 结构图

![](img/微服务结构.jpg)

> 左上角为用户使用的部分，移动端或PC端。
>
> 通过负载均衡来分配请求去哪个服务器。
>
> 试想，访问物理地址更近的服务器才能得到较快的反应速度。
>
> 再经过流控，负载均衡等才能调用到真正的服务，Consumer。
>
> Consumer去调用Provider提供的服务来完成请求。
>
> 最后将结果返回才是我们看到的结果。

![](img/微服务结构2.png)
